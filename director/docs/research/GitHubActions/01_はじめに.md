## はじめに

「戦闘力100倍プロジェクト」では、ソフトウェアの進化サイクルのコアとなる「DevOps」、「MLOps」、「不具合解析」に関してAIエージェントを用いることで、通常の100倍ほどの超効率化（ソフト外注で3ヵ月かかることを1日に短縮）を可能とする技術を開発することを目的とする。

そのうち本プロジェクト（AIエージェント活用自動ソフト開発プロジェクト）では、上記を実現するAIエージェントのフレームワークを試作し、まずは生産性の20~30倍向上（ソフト外注で1ヵ月かかることを1日に短縮）を目標とする。

本資料は、AI AgentをGitHub ActionsのClaude Codeで実行することを検討するにあたり、
GitHub ActionsでのClaude Code使用方法とEC2上でのClaudeAgentSDKとの違いについて調査した結果を記載する。

## 1. 概要調査：GitHub Actions Claude Code 調査結果

### 1.1. 目的

本調査の目的は、開発チームがGitHub ActionsでClaude Codeを効果的に活用し、
開発ワークフローの自動化と品質向上を実現するための知見を蓄積することである。具体的には以下を目指す：

- **CI/CDパイプラインとの統合**: GitHub ActionsとClaude Codeを連携させ、Issue/PRの自動処理を実現する。
- **EC2版との使い分け指針の明確化**: 各環境のメリット・デメリットを把握し、適切な使い分けを実現する。
- **コンテキスト範囲の理解**: GitHub Actions版が参照するコンテキストの範囲と制限を把握する。
- **実践的な導入指針の提供**: 即座に導入可能な設定方法と運用知見を蓄積する。

### 1.2. 参照

- [Claude Code Action - GitHub](https://github.com/anthropics/claude-code-action)
- [GitHub Actions ドキュメント](https://docs.github.com/ja/actions)
- [Claude Code ドキュメント](https://docs.claude.com/ja/docs/claude-code)
- [AWS Bedrock ドキュメント](https://docs.aws.amazon.com/bedrock/)

### 1.3. 概要

GitHub ActionsでClaude Codeを使用するにあたり、以下の観点で調査を実施した：

1. **GitHubActionsの動作の実態**: ワークフローの構成、トリガーイベント、実行環境
2. **同時並行でのワークフロー実行**: 複数Issue/PRでの同時実行時の挙動と注意点
3. **EC2上で使用するClaudeAgentSDKとの違い**: 両環境のメリット・デメリット比較
4. **参照するコンテキスト範囲**: リポジトリ、Issue、PRのコンテキストアクセス範囲
5. **GitHubActionsの使用方法**: セットアップ手順、設定ファイル、運用方法

### 1.4. 特徴

GitHub Actions版Claude Codeの主な特徴を以下に記載する。

1. GitHub統合関連
   - Issue/PR/コメントへの自動応答が可能
   - `@claude`メンションによるトリガー起動
   - ブランチ作成、コミット、プッシュの自動化
   - PR作成リンクの自動生成

2. 運用・インフラ関連
   - インフラ管理不要（サーバー保守、OS更新不要）
   - 複数Issue/PRに対する並列実行が可能
   - 実行時のみ課金（GitHub Actions + API使用分）
   - バックグラウンド実行（夜間バッチ処理が可能）

3. セキュリティ関連
   - GitHub SecretsでAPIキーや認証情報を安全に管理
   - OIDC認証でAWS認証情報の長期保存が不要
   - 実行時の権限制御（allowedToolsによる制限）
